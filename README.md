# Customer_shopping_behaviour_analysis
# Overview

This project demonstrates an end-to-end data analytics workflow using Python, SQL, and Power BI. The objective is to analyze customer shopping behavior, identify purchasing trends, and generate actionable business insights through data analysis and visualization.

The project covers the complete analytics lifecycle, including data loading, cleaning, exploratory data analysis (EDA), SQL-based business analysis, interactive dashboard creation, report documentation, and presentation development.

# Dataset
The dataset contains customer shopping transaction records, including information such as:
- Customer demographics
- Product categories
- Purchase amount
- Payment methods
- Subscription status
- Discount information
- Purchase frequency
- Customer location
- Seasonal purchases

# Tools & Technologies
- Python (Pandas, NumPy, Matplotlib, Seaborn)
- SQL (PostgreSQL/MySQL)
- Power BI
- Jupyter Notebook
- Gamma AI (Presentation)
- Microsoft Word (Project Report)

# Project Workflow
1. Data Loading
- Imported the dataset into Python.
- Explored dataset structure and data types.

2. Data Cleaning
- Checked for missing values.
- Removed duplicates.
- Corrected data types.
- Prepared the dataset for analysis.

3. Exploratory Data Analysis (EDA)
- Analyzed customer demographics.
- Identified revenue trends.
- Examined product category performance.
- Studied customer purchasing behavior.
- Compared subscription and non-subscription customers.
- Evaluated payment methods and seasonal trends.

4. SQL Analysis
Business questions were answered using SQL, including:
- Total revenue by category
- Average purchase amount
- Customer segmentation
- Revenue by gender and age group
- Purchase frequency analysis
- Discount analysis
- Subscription analysis
- Window functions and aggregate queries
